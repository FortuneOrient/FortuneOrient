如何用 Hexo3.0 打草稿，写文章

发表于 2016-07-28

本文介绍了Hexo3.0的常用命令和使用技巧。

创建文件

执行下列命令，创建新内容。

$ hexo new [layout] <title>

说明：layout 可分为三种

  • draft – 草稿，存放在source/_drafts下，不会被部署到公网
  • post – 正式文章，存放在source/_posts下，将要部署到公网
  • page – 辅助页面，存放在source下

前置格式

草稿文件，最上方以---分隔的区域，用于指定文件的变量。其中，分类（categories）和标签（tags）非常实用。举例如下：

categories:
- Diary
tags:
- PS3
- Games

具体说明，请参考Front-matter

简介与正文

建议为每篇文章编写简介。
简介应放在正文前面，二者之间通过<!--more-->隔开。
简介将出现在文章列表页，没有简介可能会输出全文。

发表草稿

草稿写好后，可通过publish命令，发表为正式文章

$ hexo publish <title>

本地预览

请确认已经正确安装了hexo-server模块。

启动服务

执行如下命令，可以启动本地临时服务器。页面地址http://localhost:4000

$ hexo server

查看草稿

如果希望启动服务时，也能看到草稿文件，需要带上参数--draft，如下：

$ hexo server --draft

关闭服务

Press Ctrl+C to stop.

生成文件

使用Hexo生成静态文件，命令：

$ hexo generate

新生成的静态文件，存放在public目录下

部署文件

如果需要把本地文件，部署到Github上，请先安装hexo-deployer-git模块：

$ npm install hexo-deployer-git --save

如何配置，请参考官网 – 部署。

配置完毕后，执行以下命令，开始部署文件。

$ hexo deploy

总结

使用Hexo写文章，基本流程如下:

  1. 新建草稿文件
  2. 写内容，写简介，设定tags…
  3. 发表草稿
  4. 本地预览 $ hexo s
  5. 生成静态文件 $ hexo g
  6. 部署到公网 $ hexo d
